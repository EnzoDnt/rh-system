# Documentation

Bienvenue. Cette doc est organisée par étapes successives — du "à quoi ça sert" jusqu'à "comment l'opérer en prod".

Choisis ton point d'entrée selon ton profil :

| Tu es… | Commence par |
|---|---|
| Curieux de comprendre ce que fait le système | [01-comprendre/vue-d-ensemble.md](01-comprendre/vue-d-ensemble.md) |
| Développeur qui veut le lancer en local en 30 min | [02-demarrer/parcours-developpeur.md](02-demarrer/parcours-developpeur.md) |
| Non-technique, tu veux le déployer en cliquant | [02-demarrer/parcours-clic-a-clic.md](02-demarrer/parcours-clic-a-clic.md) |
| Tu veux que ton agent IA fasse l'install pour toi | [02-demarrer/parcours-avec-agent-ia.md](02-demarrer/parcours-avec-agent-ia.md) |
| Tu cherches un détail technique précis | [99-reference/](99-reference/) |

## Plan complet

```
01-comprendre/      Comprendre le système (architecture, pipeline candidat, choix techniques)
02-demarrer/        Trois parcours d'installation (dev, clic-à-clic, agent IA)
03-deployer/        Choisir et exécuter le déploiement (Coolify, Vercel+Railway, VPS, local)
04-personnaliser/   Adapter le système à TON cas (prompts, scoring, branding, intégrations)
05-operer/          Faire tourner le système au quotidien (monitoring, incidents, backup)
99-reference/       Référence technique exhaustive (API, modèle de données, env vars, glossaire)
```

Si tu te perds : retour au [README racine](../README.md).
